A Colorful Guide to Linux Commands and DevOps

Essential Linux Commands

ls - List directory contents

Examples:

ls -la (list all files including hidden ones with details)

ls -lh (list with human-readable file sizes)

cd - Change directory

Examples:

cd /path/to/directory (go to specific directory)

cd .. (go up one directory)

cd ~ (go to home directory)

mkdir - Create directories

Examples:

mkdir new_directory

mkdir -p parent/child/grandchild (create nested directories)

grep - Search text using patterns

Examples:

grep "pattern" filename

grep -r "pattern" directory/ (recursive search)

find - Search for files in directory hierarchy

Examples:

find /path -name "*.txt"

find /path -type f -mtime -7 (files modified in last 7 days)

Advanced Linux Commands

sed - Stream editor for filtering and transforming text

Examples:

sed 's/old-text/new-text/g' filename

sed -i '5d' filename (delete line 5)

awk - Pattern scanning and processing language

Examples:

awk '{print $1}' filename (print first column)

awk -F: '{print $1, $3}' /etc/passwd (print columns 1 and 3)

tar - Archiving utility

Examples:

tar -cvf archive.tar files/ (create archive)

tar -xvf archive.tar (extract archive)

tar -czvf archive.tar.gz files/ (create compressed archive)

DevOps Concepts and Tools

Continuous Integration (CI)

Automatically integrate code changes from multiple contributors into a shared repository. Popular tools include Jenkins, GitLab CI, and GitHub Actions.

CI ensures that new code doesn't break existing functionality by running automated tests on every commit.

Continuous Deployment (CD)

Automatically deploy applications to production or staging environments after passing testing phases.

CD reduces manual processes and provides faster feedback loops for development teams.

Infrastructure as Code (IaC)

Manage infrastructure through code instead of manual processes. Tools like Terraform, Ansible, and CloudFormation allow versioning, reuse, and consistent deployments.

Containerization

Package applications with their dependencies into isolated units. Docker is the most popular containerization tool, providing consistency across environments.

Kubernetes

Container orchestration platform that automates application deployment, scaling, and management across clusters of hosts.

Key concepts: Pods, Services, Deployments, ConfigMaps.

Monitoring and Logging

Tools like Prometheus, Grafana, ELK Stack (Elasticsearch, Logstash, Kibana), and Datadog provide visibility into application performance and system health.

Common DevOps Commands

docker build - Build an image from a Dockerfile

docker build -t myapp:latest .

docker run - Run a container from an image

docker run -d -p 8080:80 --name mycontainer myapp:latest

kubectl - Kubernetes command-line tool

kubectl get pods

kubectl apply -f deployment.yaml

kubectl logs pod-name

git - Version control commands

git clone https://github.com/username/repo.git

git checkout -b new-branch

git commit -m "Message"

git push origin branch-name
